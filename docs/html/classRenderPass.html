)

Create a new render pass for rendering to a specific set of targets.

Paramètres
color_targetsOne or more target objects to which color information will be rendered. Must either be a Screen or a Texture instance.
depth_targetTarget object to which depth information will be rendered. Must either be NULL or a Texture instance.
stencil_targetTarget object to which stencil information will be rendered. Must either be NULL or a Texture instance. Can be identical to 'depth_target' in case the texture has the pixel format Texture::PixelFormat::DepthStencil.
blit_targetWhen rendering finishes, the render pass can (optionally) blit the framebuffer to another target (which can either be another RenderPass instance or a Screen instance). This is mainly useful for multisample antialiasing (MSAA) rendering where set of multi-sample framebuffers must be converted into ordinary framebuffers for display.
clearShould enter() begin by clearing all buffers?

◆ begin()

void RenderPass::begin ( )

Begin the render pass.

The specified drawing state (e.g. depth tests, culling mode, blending mode) are automatically set up at this point. Later changes between begin() and end() are possible but cause additional OpenGL/GLES/Metal API calls.

The Python bindings also include extra enter and exit aliases so that the render pass can be activated via Pythons 'with' statement.
