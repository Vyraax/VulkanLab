\doxysubsection{Description détaillée}
The world creates, destroys, and manages entities. The lifetime of entities and {\itshape registered} systems are handled by the world (don\textquotesingle{}t delete a system without unregistering it from the world first!), while event subscribers have their own lifetimes (the world doesn\textquotesingle{}t delete them automatically when the world is deleted).

{\footnotesize\ttfamily void E\+C\+S\+::\+World\+::destroy (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{classECS_1_1Entity}{Entity}} $\ast$}]{ent,  }\item[{bool}]{immediate = {\ttfamily false} }\end{DoxyParamCaption})\hspace{0.3cm}{\ttfamily [inline]}}

Destroy an entity. This will emit the On\+Entity\+Destroy event.

If immediate is false (recommended), then the entity won\textquotesingle{}t be immediately deleted but instead will be removed at the beginning of the next \mbox{\hyperlink{classECS_1_1World_aac33fad54c600dc5e5f477a663ede9c7}{tick()}} or when \mbox{\hyperlink{classECS_1_1World_a6a9fd886acec3cdec144a5a8e62b0d26}{cleanup()}} is called. On\+Entity\+Destroyed will still be called immediately.

This function is safe to call multiple times on a single entity. Note that calling this once with immediate = false and then calling it with immediate = true will remove the entity from the pending destroy queue and will immediately destroy it {\itshape without} emitting a second On\+Entity\+Destroyed event.

\mbox{\hyperlink{structA}{A}} warning\+: Do not set immediate to true if you are currently iterating through entities! \mbox{\Hypertarget{classECS_1_1World_a8ab7fc08cb14e9226c9ad03345ee751c}\label{classECS_1_1World_a8ab7fc08cb14e9226c9ad03345ee751c}}
